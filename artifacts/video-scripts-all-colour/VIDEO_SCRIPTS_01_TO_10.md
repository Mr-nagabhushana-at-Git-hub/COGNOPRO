# COGNO Pro Workspace Video Scripts

These scripts are written for the merged `all-colour` exports.  
Each clip opens in light mode, then transitions smoothly into dark mode while keeping the same spoken script.

## 01 Dashboard

Voiceover:
Welcome to the COGNO FocusFlow Nexus. This dashboard is the live control surface for the personal intelligence layer, where health, focus, planning, notes, fitness, and agent guidance all connect into one workspace. The interface stays cinematic, but the point is practical: every major subsystem is reachable in one move, and the whole environment is built to feel like a calm high-performance operating layer for daily execution.

Transition cue:
Hold on the central core, then fade into dark mode while the navigation dock remains the anchor.

## 02 Tasks & Matrix

Voiceover:
This is the Tasks and Matrix engine. Users can import messy files, rough notes, CSVs, or plain text, and the backend cleans the input, structures the work, and drops it into the Eisenhower matrix automatically. From there, tasks can still be dragged between quadrants, corrected manually, and pushed into Google Calendar so the system moves from capture to scheduling without friction.

Transition cue:
Show the import and action controls first, then crossfade into dark mode over the quadrant grid.

## 03 Planner

Voiceover:
The Planner view is intentionally quieter. It gives the user a clean place to reserve time, review the day, and protect focus blocks without clutter. This is where the execution layer becomes visible on a calendar rhythm, so planning is not just a list of intentions but a shaped day with real placement.

Transition cue:
Let the greeting and date land in light mode, then move into the dark mode schedule view.

## 04 Notes

Voiceover:
Notes is the fast memory surface of the system. Users can create quick notes, pin the important ones, assign visual colors, and keep context close to the rest of the workflow. Instead of treating notes as a separate app, COGNO keeps them inside the same intelligence loop so ideas, reminders, and small fragments stay available when tasks and agents need them.

Transition cue:
Begin on the composer and note grid in light mode, then crossfade into the dark note wall.

## 05 Mental Wellness

Voiceover:
The Mental Wellness space is built for reflection, journaling, stress tracking, and supportive companion interactions. It is not just a pretty panel; it records entries, analyzes patterns, highlights trigger trends, and supports calmer routines such as monk mode. The visual style is immersive, but the real goal is helping the user understand emotional state and maintain steadier performance over time.

Transition cue:
Use the 3D zen core as the visual bridge between light and dark mode.

## 06 Health Predict

Voiceover:
Health Predict combines symptom selection with body metrics such as BMI, then enriches the result with fitness context when that data exists. The system can record predictions, keep a history of results, and return agent insight instead of a flat one-shot output. This turns the page into a health reasoning workflow rather than a simple form submission screen.

Transition cue:
Show the staged wizard in light mode, then fade into the dark mode prediction flow.

## 07 Ultra Agent

Voiceover:
Ultra Agent is the personal pulse layer. It reads across the user’s tasks, reflections, calendar state, movement, focus readiness, and monk mode plan to surface a daily operating picture. The agents stay mostly in the backend, but this page shows the effect of that orchestration by turning scattered signals into a coherent protocol for what the user should do next.

Transition cue:
Anchor the transition on the pulse canvas and the daily pulse card.

## 08 Brain Training

Voiceover:
Brain Training brings cognitive exercises directly into the workspace. Instead of one isolated game, the page collects multiple training modes, performance history, and lightweight stats so users can build mental sharpness alongside the rest of their productivity and health system. It makes cognition part of the same operating loop as planning, wellness, and fitness.

Transition cue:
Open on the game grid and overall stats, then crossfade into the dark-mode training hub.

## 09 Fitness Tracker

Voiceover:
The Fitness Tracker is designed as a command center for physical progress. It supports sync, bio-profile information, live telemetry, movement metrics, and workout pathways that can later feed the health prediction layer. This is important because the system is not treating health and productivity as separate worlds; movement, recovery, and workload all inform each other.

Transition cue:
Begin with the command center header in light mode, then transition into the darker telemetry-heavy layout.

## 10 Platform Settings

Voiceover:
Platform Settings is where the real integrations and security posture come together. This page manages model orchestration, Google and health connectivity, TEE-oriented privacy messaging, and future local inference pathways such as ARM-based Gemma plans. The surface stays clean, but underneath it is the wiring layer that lets COGNO sync with real providers instead of acting like a closed demo.

Transition cue:
Start with the orchestration hero area in light mode, then fade into the dark secure configuration view.
